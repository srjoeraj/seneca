Switch100 (SwitchAA)

en
conf t
hostname Switch-100
ip domain-name csn400.com
enable password seneca
!
vtp mode client
vtp domain student77
!
int fa 0/1
switchport access vlan 100
spanning-tree portfast
spanning-tree bpduguard enable
!
int range fa 0/2 - 22
switchport access vlan 99
spanning-tree portfast
spanning-tree bpduguard enable
!
int gig 0/1
switchport mode trunk
switchport trunk allowed vlan 100,77,98,106,250
!
int range fa 0/23 - 24
channel-protocol lacp
channel-group 1 mode active
!
int po1
switchport mode trunk
switchport trunk allowed vlan 100,77,98,106,99,250
!
int vlan 250
ip address 172.16.250.100 255.255.255.0
no shut
!
ip default-gateway 172.16.250.1
!
!SSH setup script -- run this script separately!
crypto key generate rsa
1024
ip ssh version 2
line vty 0 4
transport input ssh 
login local
username student100 password csn400
username student77 password csn400
username student98 password csn400
username student106 password csn400
exit

To SSH into you Routers

  • Well If you read the scott script's you'll realise:
    • He has blocked traffic coming to port 22 on the router
    • Port 22 runs SSH, oh yes it does.
    • So I have changed the default ssh port on the router. because I could.

  • New SSH ports for the routers are:

    • Router 100: 2200
    • Router 77: 2077
    • Router 106: 2206
    • Router 98: 2298

  • So to ssh follow the syntax:

user@computer:$ ssh studentXX@123.45.56.7 -p AAAA
